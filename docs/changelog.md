# FluentFlow Changelog

本文档记录 FluentFlow 面向使用者和维护者的版本变化。它和 `docs/product_growth_log.md` 的分工不同：

- `product_growth_log.md` 记录为什么改、做过哪些判断和取舍。
- `changelog.md` 记录这次改了什么、影响什么用户、上线时要注意什么。

每次有可感知的产品变化、部署变化、数据口径变化或破坏性调整时，都应该补一条。不要把还没落地的计划写成已发布能力。

## 记录格式

```md
## YYYY-MM-DD｜一句话版本主题

### 用户可见变化
- ...

### 维护者变化
- ...

### 数据 / 口径变化
- ...

### 注意事项
- ...
```

如果某一类没有变化，可以省略。注意事项里要写清是否需要重启后端、重新构建前端、迁移数据、更新环境变量或通知用户。

## Unreleased

后续尚未准备发布的变更继续记录在这里。只有已经落地、验证并准备对外说明的内容，才移动到具体版本段落。

### 用户可见变化

- 视频链接任务失败后会保留为后台失败任务，开始处理页和首页会提示“已保存在后台任务”，后台任务卡片会持续展示失败原因。
- 视频链接提交成功后会自动进入任务处理详情页；实时进度、处理阶段和 Agent 判断统一在同一处查看，输入页不再保留链接任务进度卡片。
- 任务处理详情页移除重复的执行记录模块，顶部负责实时进度，页面主体聚焦 Agent 判断、证据和下一步建议。
- 本地执行的 YouTube 字幕链接任务会在抓到字幕后继续生成笔记，不再因为后台转交字幕文件时缺少本地执行标记而被账号登录拦截。

## v0.2.0｜2026-06-29｜Agent 工作流与云端转录基础版

### 用户可见变化

- 云端转录主路线切换到 ElevenLabs Scribe；Azure Batch 仅保留为 legacy 兼容路径。
- 开始处理页支持在已有转录任务运行时继续添加新音视频进入后台队列。
- 后台任务页统一展示账号任务、本机任务、失败任务和已取消任务，并支持删除已完成、失败或已取消记录。
- 视频链接任务会在主页和后台任务中显示解析、下载、保存、转录和摘要进度，标题优先展示真实视频标题。
- 编辑器支持“重生笔记”和“重新转录”二次确认，避免误触覆盖当前笔记或转录。
- 编辑器新增文本校对 / 视频复查切换；有原视频和时间戳时，可在视频下方编辑当前字幕并跳转上一句 / 下一句。
- 编辑器左侧可保存并复用播放音频，刷新后仍可恢复音频对照和重新转录来源。
- 英文材料默认保留英文原文转录，并可在翻译成功时生成中英双语字幕；摘要仍直接生成中文笔记。
- 字幕导入明确为“导入字幕生成笔记”，支持 SRT、VTT、TXT 和 Markdown，并纳入账号额度与提交频率保护。
- AI 笔记自动选择接入真实规划，会基于材料统计、开头、中段和结尾样本判断直接生成或高保真笔记。
- 长材料新增章节覆盖笔记能力：先抽证据、规划章节、逐章生成，再做覆盖检查，适合课程和讲座类材料。
- 笔记正文支持显示 Agent 选中的视频关键截图，并可隐藏或显示截图，减少对正文阅读的干扰。
- 视频关键截图会经过密度、重复画面和低价值页面过滤；封面、目录、纯标题页和过渡帧不会挤占笔记。
- 下载 Markdown、PDF、Word 时会保留已选截图；飞书 OpenAPI 导出会尝试把本地截图上传为文档图片块。
- Agent 工作流页从旧处理设置表单改为任务解释页，展示当前素材的材料判断、路线选择、笔记策略、影响和复查点。
- Agent 接入面板支持创建、查看和撤销 API Key；新建 Key 会自动填入 Claude Code / Codex 的 MCP 配置示例。
- 本地和云端 Agent/MCP 接入统一使用 API Key，明文 Key 只在创建成功后显示一次。
- 编辑器 AI 摘要底部下线“生成详情”和重复元信息，解释入口收敛到“查看 Agent 工作流”。
- 登录 / 访问门禁、关于与协议、提示词模板弹窗和设置维护区统一到当前 FluentFlow 中性工作流 UI 语言。
- 左下角“关于与协议”补齐服务条款、隐私政策和版本更新入口；版本更新页读取项目 changelog 并展示具体条目。
- 关于与协议页面改为自带滚动容器，长内容不会再被应用根层锁住。
- 首页主标题从“今天想创作些什么呢”改为“今天想记录些什么呢”，更贴近转录、整理和复查学习资料。
- 输入视频链接后的当前任务标题会先显示平台和视频 ID，不再把 B 站跟踪参数当标题；取消任务按钮文案更明确。
- Bilibili 链接解析不再使用 miuistore 第三方兜底；解析失败时提示上传本地视频或配置浏览器 cookies 后重试。
- YouTube 链接会优先抓取可用字幕生成笔记，避免长视频下载阶段出现 403、429 或超时后无法产出笔记。
- 视频转写与总结页顶部操作区在较窄宽度下保持单行排布，避免模式切换和查看后台按钮互相挤压。

### 维护者变化

- 新增 `/agent/v1` Agent 数据链路，支持提交链接/转录文本、等待任务、获取任务包、读取诊断、重生笔记和导出飞书。
- 新增本地 MCP Server 和检查脚本，让 Claude Code、Codex 等外部 Agent 可以通过稳定 API 操作 FluentFlow。
- 新增 SQLite API Key 存储，服务端只保存哈希，并允许 `X-FluentFlow-Access-Token` 或 `Authorization: Bearer` 携带 Key 调用 `/agent/v1`。
- `/agent/v1` 纳入后端 API 路由前缀和账号鉴权边界，API Key 会绑定到账号 scope 或本地 client scope。
- 新增 Agent / MCP parity 规则：未来修改用户工作流时，需要同步判断 Agent API、MCP 工具、任务包字段、schema、文档和测试是否更新。
- 前端任务/历史映射抽到 `frontend/src/lib/jobMappers.js`，设置、转录路线和云端配置判断抽到 `frontend/src/lib/settingsModel.js`。
- Agent 任务的“重生笔记”和“导出飞书”动作抽到 `backend/core/agent_task_actions.py`，router 只保留请求入口。
- 本地/云端执行通道判断拆到后端 request scope 和前端 local execution 模块，减少 path/header 规则复制。
- 前端新增 api client 与 task state 口径，`cached_only` 成为正式前端任务状态，旧 `__cacheOnly` 只作为历史兼容输入。
- 本地运行产物默认迁移到系统应用数据目录，并新增显式迁移脚本和统一 runtime path 管理。
- 后台任务队列改为持久化任务步骤模型，上传、访客试用和视频链接任务先写入可恢复步骤，再由 worker 领取执行。
- 视频链接任务新增 YouTube 字幕优先路径和资产策略 metadata：`yt-dlp` 逐个字幕语言尝试，成功后走现有 `/summarize-transcript-file` 链路；无字幕时才回退媒体下载，并按视频时长/估算大小调整下载预算。
- 新增取消任务、删除任务、播放音频补写、中文翻译补写等后端接口，支撑编辑器和后台任务恢复动作。
- 新增离线笔记质量评测、STT 候选比较、MCP 检查和日常 change-control 检查脚本。
- 新增产品版本管理基础：`VERSION` 作为唯一版本源，后端 `/version` 和前端构建配置暴露版本、Git commit 与 schema 信息。
- 新增 release prepare / release check 流程、release checklist 和版本策略文档，区分日常 checkpoint 与正式 release gate。
- 补充关于二级页面 UI、一致性文案、Agent 工作流说明、结果 schema、MCP 集成、运维和回归检查的维护文档。

### 数据 / 口径变化

- 任务结果新增 `result_schema_version = "2"`；新写入保存长期字段 `raw_segments` / `display_segments`。
- `segments`、`bilingual_segments`、`translated_segments_zh`、`cleaned_segments` 等旧字段只作为历史兼容读取入口。
- 任务结果新增 `processing_plan`、`source_language`、`subtitle_mode`、`translation_status`、提示词模板元数据和 note mode plan 元数据。
- 视频来源 metadata、结果 JSON 和本地历史条目新增 `raw_title` / `display_title`；`source_filename` 继续保留为机器存储文件名。
- 新增 SQLite `job_steps` 表；`jobs` 继续作为用户可见任务状态，`job_steps` 作为后台 worker 的执行队列。
- 飞书导出新增长期字段 `lark_export_route` / `larkExportRoute`，旧 `lark_via_cli` / `larkViaCli` 继续兼容。
- 热词审阅旧能力长期下线，相关端点保留 410 Gone 墓碑响应；本机历史候选扫描与账号导入旧路线也保留 410 兼容端点。

### 注意事项

- 升级需要重新构建前端并重启后端；浏览器如仍命中旧 bundle，需要强制刷新。
- 生产环境需要确认 `FLUENTFLOW_ALLOWED_STT_PROVIDERS`、`FLUENTFLOW_DEFAULT_STT_PROVIDER`、ElevenLabs API Key、飞书 OpenAPI 凭证和对象存储/截图配置。
- 本版本包含 result schema、任务步骤表、运行产物路径和历史标题语义变化；旧数据应通过兼容读取和启动补齐逻辑继续可读。
- 正式部署、打 tag 和发布 manifest 仍需在 clean worktree 上按 release checklist 执行。

## 2026-06｜公开试用前的账号、隔离与额度边界

### 用户可见变化

- 支持账号模式，云服务器公开试用时建议使用账号登录，而不是只靠访问码。
- 未启用账号模式时，前端会生成设备级 `client_id`，不同浏览器或设备默认看不到彼此的历史任务。
- 云服务器公开模式下，普通用户不需要理解本地 STT、Azure Key、Blob SAS、pyannote token、飞书 App Secret 等维护者配置。

### 维护者变化

- 增加上传大小、队列数量、单设备活跃任务数、全站活跃任务数、每日任务数、每日上传量和 IP 提交频率限制。
- 增加部署自检脚本和清理脚本，用于上线前检查配置、上线后控制磁盘增长。
- 云服务器推荐使用 Azure Batch 作为默认 STT Provider；本地 faster-whisper 保留为开发和本地使用路径。

### 数据 / 口径变化

- 任务历史、下载、转录稿编辑和取消任务会按账号或设备作用域隔离。
- 设备级隔离不是正式账号系统；用户清空浏览器数据或换浏览器后，会变成新的设备身份。

### 注意事项

- 公开试用建议开启 `FLUENTFLOW_AUTH_MODE=accounts`。
- 如果只启用访问码或设备级隔离，不要对外承诺“账号级历史同步”。

## 2026-06｜Azure Batch 云转录主路径

### 用户可见变化

- 支持通过云端 Azure Batch 处理长音视频转录。
- 任务页会展示上传 Blob、提交任务、等待 Azure、下载结果等阶段。
- Azure Batch 阶段不伪造 STT 百分比，只展示真实阶段和等待状态。

### 维护者变化

- 需要配置 Azure Speech 地址、Key 和 Blob container SAS URL。
- 音视频会先压缩为 MP3，再上传到 Blob，避免本地 Whisper 专用 WAV 体积膨胀。
- 提供 `scripts/check_azure_batch_transcription.py` 做 dry-run 和真实 smoke test。

### 数据 / 口径变化

- `stt_completed.metadata.stt_provider` 可区分本地 faster-whisper、Azure Batch 和历史 Azure Fast。
- STT 性能报告必须按 provider、设备、模型、速度档和语言口径分组。

### 注意事项

- Azure Batch 的等待时间包含上传、网络、云端排队和处理时间，不能和本地推理耗时直接混成一个平均数。

## 2026-06｜转录编辑与修改记录沉淀

### 用户可见变化

- 转录结果页支持编辑分段文本。
- 用户修改后的转录稿会自动保存，避免人工校对结果丢失。
- 修改记录会沉淀为本地资产，包含修改前后文本和相邻上下文，便于后续 STT 质量评估。

### 维护者变化

- 编辑稿和修改记录保存在本地数据目录中，属于用户内容，不应提交到 Git。
- 清理策略需要同时考虑源文件、产物、编辑稿和修改记录。

### 数据 / 口径变化

- 转录编辑记录可以用于质量评估和错误类型分析，但不能直接包装成“用户满意度”或“笔记可用率”。

### 注意事项

- 目前没有完整人工校对计时入口，不应从编辑行为推断校对完成耗时。

## 2026-06｜长字幕高保真笔记生成

### 用户可见变化

- 笔记生成支持自动选择、直接长上下文和高保真笔记三种模式。
- 长字幕会先剥离 SRT/VTT 时间码、序号和标签，再按内容段落重组，减少无关格式占用上下文。
- 高保真模式会先分段提取证据，再生成终稿，适合长课程和信息密度高的材料。

### 数据 / 口径变化

- 摘要事件会记录请求模式、实际采用模式、分段数和覆盖检查信息。
- `summary_completed` 只代表 AI 摘要真实成功，不代表最终质量一定可用。

### 注意事项

- 不要把“下载摘要”或“导出飞书”写成“用户认可笔记质量”。

## 2026-06｜第一阶段事件日志

### 维护者变化

- 新增 best-effort SQLite 事件日志：`data/fluentflow_events.sqlite`。
- 提供事件导出脚本，支持 JSON / CSV。

### 数据 / 口径变化

- 记录真实发生的导入、音频提取、STT、转写就绪、摘要完成/失败/跳过、飞书导出、下载、取消和任务终态。
- 不保存完整转写文本或完整 Markdown，只保存长度、耗时、状态、错误原因和链接。
- 不记录当前产品中没有真实入口的行为，例如评分、校对完成、笔记可直接使用。

### 注意事项

- 历史 localStorage 快照不能写入事件表，不能伪造成未来事件日志。

## 2026-06｜本地 STT 可取消与真实进度口径

### 用户可见变化

- 本地 faster-whisper 转录任务支持真正取消，取消后会终止后端 STT 子进程。
- 转录阶段展示模型加载、准备音频、等待第一段、接收片段等状态。
- 第一段 segment 产出前不伪造进度百分比。

### 维护者变化

- STT 从线程池迁移到独立子进程，牺牲部分模型热缓存速度，换取资源可控。

### 数据 / 口径变化

- `stt_completed` 记录设备环境、模型、速度档、语言、实时倍率和模型加载耗时。
- 本地 STT 耗时必须说明设备和模型口径。

### 注意事项

- SSE 连接断开不等于用户取消任务；用户点击取消才是明确取消行为。
