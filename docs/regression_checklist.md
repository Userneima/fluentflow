# FluentFlow 回归检查清单

本文档用于每次修改、部署或公开试用前的回归验证。目标不是追求形式完整，而是避免把已经修好的核心体验改坏。

使用方式：

- 小改动：跑相关模块的 P0 检查。
- 前端交互或任务链路改动：跑全部 P0。
- 上线、部署、账号、队列、Azure、文件保存相关改动：跑 P0 + P1。
- 发现失败项时，先记录现象和复现路径，再修复；不要只刷新页面确认一次。

## P0：必须通过

### 1. 基础启动

| 检查项 | 操作 | 通过标准 |
| --- | --- | --- |
| 前端构建 | `npm run build:frontend` | 无报错，`frontend/assets/app.js` 和 CSS 可更新 |
| 后端导入 | `python3 -m compileall backend scripts tests` | 无语法错误 |
| 自动测试 | `PYTHONPATH=. ./venv/bin/pytest -q` | 现有测试通过 |
| 首页可打开 | 访问 `http://127.0.0.1:8000` | 页面正常加载，无白屏 |

### 2. 账号 / 访问 / 历史隔离

| 检查项 | 操作 | 通过标准 |
| --- | --- | --- |
| 未登录访问 | 开启账号模式后打开页面 | 应进入登录或注册流程 |
| 首个账号 | 新库首次注册 | 第一个账号成为管理员 |
| 历史隔离 | 用两个浏览器或两个账号分别处理任务 | 互相看不到对方任务 |
| 设备级隔离 | 未开启账号模式时，用不同浏览器访问 | 任务历史按浏览器设备隔离 |
| 退出 / 换账号 | 退出后重新登录另一个账号 | 历史列表切换到当前账号作用域 |

### 3. 音视频上传到结果页

| 检查项 | 操作 | 通过标准 |
| --- | --- | --- |
| 上传小视频 | 上传 1-3 分钟 MP4 | 进入任务卡片，显示真实阶段 |
| STT 完成 | 等待转写结束 | 打开结果页，能看到分段转录 |
| 摘要完成 | 不开启仅转录 | 能看到 AI 摘要，不把失败 fallback 成成功摘要 |
| 任务终态 | 查看最近活动或任务页 | 显示已完成，耗时和文件信息合理 |
| 事件日志 | 导出事件 | 有 `source_imported`、`transcript_ready`、`task_completed` 等真实事件 |

### 4. Azure Batch 云转录

| 检查项 | 操作 | 通过标准 |
| --- | --- | --- |
| 配置预检 | 运行部署自检或 Azure dry-run | 缺配置时给出明确失败原因 |
| 上传 Blob | 上传真实音视频 | 阶段显示上传云端音频，不暴露 SAS |
| 等待 Azure | Batch 处理中 | 显示等待状态和已用时间，不伪造百分比 |
| 下载结果 | Azure 成功后 | 转成统一 segments 和转录文本 |
| 失败提示 | 故意使用错误配置或过期 SAS | 前端显示可理解原因，metadata 保留原始错误 |

### 5. 字幕 / 文本导入

| 检查项 | 操作 | 通过标准 |
| --- | --- | --- |
| 导入 SRT | 上传 SRT 文件 | 跳过音频提取和 STT |
| 导入 TXT | 上传 TXT 文件 | 直接进入摘要生成 |
| 时间码处理 | 检查摘要输入效果 | AI 使用去时间码、重组段落后的正文 |
| 事件口径 | 导出事件 | 不应出现伪造的 `audio_extracted` 或 `stt_completed` |

### 6. 转录编辑器

| 检查项 | 操作 | 通过标准 |
| --- | --- | --- |
| 分段显示 | 打开完整转录 | 左侧固定时间戳，右侧可编辑文本 |
| 修改文本 | 改一段内容 | 显示已修改和已保存状态 |
| 刷新恢复 | 刷新页面 | 修改后的文本仍在 |
| 本地保存 | 检查产物或接口 | 编辑稿已持久化，不只存在前端状态 |
| 修改记录 | 修改多段文本 | 修改记录数量稳定，不应先加一再突然清零 |
| 播放控制 | 修改时音频正在播放 | 不应把播放进度跳回 0 |

### 7. 下载与导出

| 检查项 | 操作 | 通过标准 |
| --- | --- | --- |
| 下载转录 TXT | 点击导出转录文本 | 文件内容对应当前编辑稿 |
| 下载 SRT/VTT | 从产物入口下载 | 时间戳存在，文本不空 |
| 下载摘要 | 点击下载摘要 | Markdown 文件不包含提示词正文 |
| 手动飞书导出 | 点击导出到飞书 | 成功时返回 URL，失败时给出明确原因 |
| 自动飞书导出 | 开启自动导出后处理任务 | metadata 区分 `trigger=auto` |

### 8. 取消任务

| 检查项 | 操作 | 通过标准 |
| --- | --- | --- |
| 本地 STT 取消 | 本地转录中点击取消 | 后端 STT 子进程终止，任务进入 cancelled |
| Azure 任务取消 | Azure 等待中点击取消 | 前端任务状态取消；不承诺强杀外部 Azure HTTP 请求 |
| 事件口径 | 导出事件 | 有 `task_cancelled` 和终态，不把取消记成完成 |

## P1：上线前建议通过

### 1. 多文件 / 队列

| 检查项 | 操作 | 通过标准 |
| --- | --- | --- |
| 多文件上传 | 一次选择多个文件 | 超过限制时前端或后端明确提示 |
| 活跃任务限制 | 连续提交多个长任务 | 超过单用户或全站限制时被拦截 |
| 任务恢复 | 刷新页面 | 运行中任务仍可看到状态或重新订阅 |

### 2. 链接导入

| 检查项 | 操作 | 通过标准 |
| --- | --- | --- |
| 抖音分享文本 | 粘贴分享文本 | 能解析、下载或给出明确失败原因 |
| 视频直链 | 粘贴可访问视频链接 | 能进入后台任务 |
| 下载失败 | 粘贴不可访问链接 | 不崩溃，不产生假完成任务 |

### 3. 摘要模式

| 检查项 | 操作 | 通过标准 |
| --- | --- | --- |
| 自动模式 | 中短转录稿 | 选择直接或合适模式，摘要成功 |
| 高保真模式 | 长转录稿 | 分段生成，不明显漏掉主要结构 |
| 仅转录模式 | 开启 skip summary | 记录 `summary_skipped`，不调用摘要成功事件 |
| 重新生成 | 结果页点击重新生成 | 记录 `summary_regenerated`，不代表质量提升 |

### 4. 事件和报告

| 检查项 | 操作 | 通过标准 |
| --- | --- | --- |
| 导出 JSON | `python3 scripts/export_events.py --format json` | 能导出可解析 JSON |
| 导出 CSV | `python3 scripts/export_events.py --format csv` | 表头完整，metadata 可读 |
| STT 报告 | `python3 scripts/report_stt_performance.py --format md` | 按 provider / 设备 / 模型分组 |
| 历史快照 | `python3 scripts/history_snapshot.py` | 不把历史快照写入事件表 |

### 5. 清理脚本

| 检查项 | 操作 | 通过标准 |
| --- | --- | --- |
| dry-run | `python3 scripts/cleanup_storage.py` | 只展示将清理内容，不删除 |
| apply 前确认 | 人工检查输出 | 不包含账号库、事件库、任务库 |
| apply 后抽查 | `--apply` 后打开最近任务 | 最近任务产物仍可访问 |

## 常见误判提醒

- HTTP 200 不等于任务成功，任务成功以业务事件和 `task_completed.final_status` 为准。
- 下载摘要不等于笔记可用。
- 导出飞书不等于用户认可质量。
- 用户修改转录稿不等于完成校对，除非未来有明确“开始校对 / 完成校对”入口。
- 设备级 `client_id` 不等于正式账号系统。
- Azure Batch 等待时间不能和本地 faster-whisper 推理时间混算。
- 字幕文件跳过 STT 是正确路径，不应补伪造事件。
